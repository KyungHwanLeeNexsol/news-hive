# NewsHive 아키텍처 개요

> 마지막 업데이트: 2026-03-26

## 프로젝트 소개

NewsHive는 **한국 주식 투자자를 위한 섹터 기반 뉴스 인텔리전스 플랫폼**이다.
개별 종목 뉴스뿐 아니라 산업 섹터 단위의 뉴스를 자동 수집·분류하여, 투자자가 간접적이지만 중요한 뉴스를 놓치지 않도록 돕는다.

예시: 대창단조(포크레인 하부)에 투자 중인 경우, 같은 섹터의 현대제철 뉴스도 자동으로 포착한다.

## 핵심 설계 패턴

### 1. 멀티 프로바이더 AI 폴백 체인

단일 AI 제공자 의존 위험을 제거하기 위해 3단계 폴백 구조를 사용한다.

```
Groq (1순위, 무료·고속) → Gemini (2순위) → OpenRouter (3순위, 최종 폴백)
```

`ai_client.py`의 `ask_ai()` 함수를 통해 모든 AI 호출이 이 체인을 거친다.

### 2. 스케줄러 기반 비동기 파이프라인

APScheduler의 BackgroundScheduler가 7개 작업을 관리한다.
각 작업은 별도 스레드에서 `asyncio.run()`으로 비동기 함수를 실행하며, uvicorn의 메인 이벤트 루프와 충돌하지 않는다.

### 3. 퍼지 중복 제거 (Fuzzy Deduplication)

뉴스 제목의 URL 기반 단순 중복 제거를 넘어, 문자 bigram 유사도(Jaccard ≥ 0.6)를 이용해 동일 기사의 변형 제목도 필터링한다.

### 4. 키워드 + AI 하이브리드 분류

1단계로 키워드 인덱스로 빠르게 종목·섹터를 매칭하고, 2단계로 AI가 관련성과 감성(긍정/부정/중립)을 정밀 분류한다.

### 5. CORS + 리버스 프록시

프론트엔드(`next.config.ts`)에서 `/api/*` 경로를 FastAPI로 프록시하므로, 브라우저는 단일 오리진에서 동작한다.

## 주요 아키텍처 결정 사항

| 결정 | 이유 |
|------|------|
| FastAPI + SQLAlchemy | 비동기 지원, Pydantic 통합, 빠른 개발 속도 |
| APScheduler BackgroundScheduler | uvicorn 메인 루프와 충돌 없이 별도 스레드에서 주기적 실행 |
| PostgreSQL | 배열 타입(`keywords TEXT[]`), 관계형 쿼리, 프로덕션 안정성 |
| Next.js App Router | RSC 지원, 파일 기반 라우팅, Vercel 무료 배포 |
| 멀티 AI 프로바이더 | 무료 티어 할당량 분산, 장애 내성 확보 |
| Oracle Cloud VM | 무료 Always Free 티어 (백엔드 호스팅 비용 $0) |
| Vercel | 프론트엔드 CI/CD 자동화, 글로벌 CDN |
